## Guidelines

- Always pay attention to clean code
- Follow the PEP 8 style guide for Python code
- Use type hints for function parameters and return values
- Use descriptive variable and function names
- Use docstrings for all functions and classes
- Use list comprehensions and generator expressions where appropriate
- Use context managers for resource management
- Use the `with` statement for file operations
- Use the `if __name__ == '__main__':` idiom for script execution
- Use the `try-except` blocks for error handling
- Use the `logging` module for logging
- Use the `argparse` module for command-line argument parsing
- Use the `unittest` module for unit testing
- Use the `pytest` module for testing
- Use the `black` module for code formatting
- Use the `flake8` module for code linting
- Use the `mypy` module for type checking
- Use the `pylint` module for code analysis
- Use the `pre-commit` module for pre-commit hooks
- Use the `tox` module for testing in multiple environments
- Use the `virtualenv` module for virtual environments
- Use the `pipenv` module for dependency management
- Use the `poetry` module for dependency management
- Use the `pip` module for package management
- Use the `docker` module for containerization
- Use the `kubernetes` module for orchestration
- Use the `ansible` module for configuration management
- Use the `terraform` module for infrastructure as code
- Use the `aws` module for AWS services
